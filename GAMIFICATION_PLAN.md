# Duolingo/말해보카 스타일 게이미피케이션 전환 계획

> **목표**: 현재 문서 + 퀴즈 수준의 인터랙션을 Duolingo/말해보카 수준의 게이미피케이션 학습 경험으로 전환
> **제약**: GitHub Pages 정적 호스팅 (서버 없음), localStorage 기반 데이터
> **참고 앱**: Duolingo (스트릭, XP, 리그, 하트), 말해보카 (문장 조립, 망각곡선 복습, 퀘스트)

## 1. Duolingo/말해보카 핵심 UX 패턴 분석

### 1-1. Duolingo 핵심 메커니즘

| 메커니즘 | 동작 방식 | 심리적 효과 |
|---------|----------|------------|
| **Streak (연속 학습)** | 매일 1개 이상 레슨 완료 시 연속일 +1, 놓치면 리셋 | 손실 회피 (loss aversion) — 스트릭 유지를 위해 매일 접속 |
| **XP (경험치)** | 레슨/퀴즈 완료 시 XP 획득, 정답률에 따라 보너스 | 즉각적 보상 — 학습 행동에 대한 즉시 피드백 |
| **Hearts (하트)** | 5개 하트, 오답 시 -1, 소진 시 레슨 중단 | 긴장감 — 집중력 향상, 실수 비용 인식 |
| **League (리그)** | 주간 XP 순위, Bronze → Diamond 10개 리그 | 사회적 경쟁 — 순위 상승 욕구 |
| **Skill Tree** | 선형 학습 경로, 이전 단계 완료해야 다음 해금 | 구조화된 진행감 — 명확한 목표 |
| **레슨 내 Progress Bar** | 레슨 중 상단에 진행률 표시 | 완료 욕구 — 거의 다 했다는 느낌 |
| **즉각 피드백** | 정답: 초록색 + 효과음, 오답: 빨간색 + 해설 | 강화 학습 — 즉시 교정 |
| **Achievement Badge** | 마일스톤 달성 시 뱃지 부여 (예: 첫 7일 스트릭) | 수집 욕구 — 모든 뱃지 획득 목표 |

### 1-2. 말해보카 핵심 메커니즘

| 메커니즘 | 동작 방식 | React Native 학습 적용 |
|---------|----------|----------------------|
| **문장 조립** | 흩어진 단어 블록을 드래그하여 올바른 문장 완성 | 코드 토큰을 드래그하여 올바른 코드 완성 |

## 2. 도입할 기능 상세 설계

### 2-1. XP (경험치) 시스템

**개요**: 모든 학습 활동에 XP를 부여하여 즉각적 보상감을 제공한다.

**XP 획득 규칙**:

| 활동 | XP | 보너스 조건 |
|------|-----|-----------|
| 레슨(문서) 읽기 완료 | +10 XP | — |
| 퀴즈 정답 (1문제) | +5 XP | 첫 시도 정답 시 +3 XP 보너스 |
| 코드 조립 정답 | +8 XP | 힌트 미사용 시 +4 XP 보너스 |
| 출력 예측 정답 | +6 XP | — |
| 버그 찾기 정답 | +10 XP | — |
| Phase 전체 완료 | +50 XP | 전 문제 정답 시 +30 XP 보너스 |
| 일일 목표 달성 | +20 XP | — |
| 복습 문제 정답 | +3 XP | 연속 정답 시 콤보 보너스 (×1.5) |

**레벨 시스템**:

| 레벨 | 필요 XP | 칭호 |
|------|---------|------|
| 1 | 0 | 🌱 새싹 개발자 |
| 2 | 100 | 📗 입문자 |
| 3 | 300 | 📘 학습자 |
| 4 | 600 | 📙 숙련자 |
| 5 | 1000 | 📕 전문가 |
| 6 | 1500 | 🏆 마스터 |
| 7 | 2500 | 💎 그랜드마스터 |

> **참고**: 레벨은 칭호/뱃지 전용이며, Phase 해금과는 무관합니다. Phase 해금은 Skill Tree(섹션 2-9)의 단일 조건으로만 관리됩니다.

**XP 애니메이션**:
- 획득 시: 화면 중앙에 "+5 XP" 텍스트가 위로 떠오르며 페이드아웃 (CSS animation)
- 레벨업 시: 화면 전체 축하 오버레이 + confetti + 새 칭호 표시

**UI 위치**: 상단 ProgressBar 오른쪽에 XP 카운터 + 레벨 뱃지 표시

```
┌──────────────────────────────────────────────────────────────┐
│ ██████████████░░░░░░░░░░  14/39 (35%)    📘 Lv.3 · 423 XP  │
└──────────────────────────────────────────────────────────────┘
```

### 2-2. Streak (연속 학습) 시스템

**개요**: 매일 학습을 지속하도록 연속일을 추적한다.

**규칙**:
- 하루에 최소 1개 활동 (퀴즈 1문제 or 레슨 1개 완료)을 하면 스트릭 +1
- 날짜 기준은 로컬 타임존의 자정 (00:00)
- 하루라도 빠지면 스트릭 리셋 → 0

**스트릭 판정 로직 (접속 시 실행)**:
1. 접속 시 `lastActiveDate`와 오늘 날짜를 비교
2. **오늘과 같은 날** → 이미 활동 기록됨, 변경 없음
3. **어제** → 정상 (오늘 첫 활동 시 스트릭 +1)
4. **2일 이상 차이** → Streak Freeze 보유 시 빠뜨린 날 수만큼 소모하여 보호, 부족하면 스트릭 리셋
5. 판정 결과를 즉시 localStorage에 저장

> 정적 사이트이므로 사이트를 열지 않으면 JS가 실행되지 않습니다. 따라서 스트릭 판정은 항상 **다음 접속 시점**에 소급 처리됩니다.

**Streak Freeze (스트릭 보호)**:
- 3일 연속 학습 달성 시 Streak Freeze 1개 자동 획득 (최대 2개 보유)
- 접속 시 빠뜨린 날이 감지되면 자동 사용되어 스트릭 유지
- 사용 시 "🛡️ Streak Freeze로 스트릭이 보호되었습니다!" 알림

**UI**: 사이드바 상단에 불꽃 아이콘 + 연속일 표시

```
┌─────────────────────┐
│  🔥 12일 연속 학습!   │
│  ░░░░░░░░░░░░░░░░░  │
│  오늘 목표: 2/5 완료  │
│  🛡️ × 1 보유 중      │
└─────────────────────┘
```

**스트릭 달성 보상**:

| 스트릭 | 보상 |
|--------|------|
| 3일 | 🛡️ Streak Freeze × 1 |
| 7일 | 🏅 "일주일 전사" 뱃지 + 50 XP |
| 14일 | 🏅 "2주 마라토너" 뱃지 + 100 XP |
| 30일 | 🏅 "한 달의 집념" 뱃지 + 200 XP |
| 100일 | 🏅 "백일장" 뱃지 + 500 XP |

### 2-3. Heart (하트) 시스템

**개요**: 레슨 내 연습 문제에서 틀릴 수 있는 횟수를 제한하여 집중도를 높인다.

**규칙**:
- 레슨(문서) 진입 시 ❤️ × 5 부여
- 연습 문제 오답 시 -1 ❤️
- 하트 전부 소진 시:
  - 해당 레슨 연습 문제 잠금
  - "복습으로 하트 회복" 또는 "30분 후 재도전" 선택지 제시
  - 30분 타이머 구현: `hearts`에 `lockedUntil` ISO 타임스탬프 저장 (예: `"2026-03-09T12:30:00"`), 접속 시 현재 시각과 비교하여 잠금 해제
- 복습 문제 1개 정답 시 ❤️ +1 회복
- 하트는 레슨 단위로 리셋 (다른 레슨 이동 시 다시 5개)

**UI**: 레슨 내 연습 모드 진입 시 우측 상단에 하트 표시

#### 유형 A: 코드 블록 조립 (말해보카 문장 조립 방식)

**개요**: 흩어진 코드 토큰을 드래그하여 올바른 코드를 완성한다.

**동작**:
1. 문제: "다음 요구사항을 만족하는 React Native 코드를 조립하세요"
2. 코드 토큰들이 무작위로 섞여서 제시됨
3. 사용자가 토큰을 드래그하여 답란에 순서대로 배치
4. 제출 시 정답 여부 판정 + 피드백

**UI 와이어프레임**:
```
┌──────────────────────────────────────────────────┐
│  📝 "버튼 클릭 시 카운트를 증가시키는 코드를 조립하세요" │
├──────────────────────────────────────────────────┤
│                                                  │
│  답란 (드래그하여 배치):                             │
│  ┌──────────────────────────────────────────┐    │
│  │ [const]  [setCount]  [(prev) => prev + 1] │    │
│  │ [       아직 비어있는 슬롯...              ] │    │
│  └──────────────────────────────────────────┘    │
│                                                  │
│  사용 가능한 토큰:                                  │
│  ┌────────┐ ┌──────────┐ ┌───────────┐          │
│  │ count  │ │ useState  │ │ setCount  │          │
│  └────────┘ └──────────┘ └───────────┘          │
│  ┌────────────────────┐ ┌──────┐ ┌─────┐        │
│  │ (prev) => prev + 1 │ │  (0) │ │  =  │        │
│  └────────────────────┘ └──────┘ └─────┘        │
│  ┌────────┐ ┌────────────┐                       │
│  │ const  │ │ useEffect  │  ← 오답 유도 토큰       │
│  └────────┘ └────────────┘                       │
│                                                  │
│  [💡 힌트 보기]                    [제출하기 →]     │
└──────────────────────────────────────────────────┘
```

**마크다운 문법**:
````markdown
```exercise
type: code-arrange
question: "useState를 사용하여 count 상태를 선언하는 코드를 조립하세요"
tokens:
  - "const"
  - "[count, setCount]"
  - "="
  - "useState"
  - "(0)"
distractors:
  - "useEffect"
  - "let"
  - "([])"
answer: ["const", "[count, setCount]", "=", "useState", "(0)"]
hint: "useState는 [현재값, 설정함수] 배열을 반환합니다"
xp: 8
```
````

**정답 비교 규칙**: answer를 토큰 배열로 정의하고, 사용자가 배치한 토큰 배열과 **순서 일치 비교**한다. 토큰 간 공백/구분은 UI에서 자동 처리하며, 비교 시에는 토큰 문자열의 trim 값만 사용한다.

**구현**: 드래그 앤 드롭은 HTML5 Drag and Drop API 또는 터치 환경 대응을 위해 pointer events 기반으로 구현. 모바일에서는 탭하여 선택 → 탭하여 배치 방식.

#### 유형 B: 출력 예측

**개요**: 코드를 읽고 실행 결과(console.log 출력)를 예측한다.

**동작**:
1. 코드 스니펫 제시
2. 4개 보기 중 올바른 출력 선택
3. 정답 후 "실제로 실행해보기" 버튼으로 Playground 연결

**마크다운 문법**:
````markdown
```exercise
type: output-predict
question: "다음 코드의 console.log 출력은?"
code: |
  const [count, setCount] = useState(0);
  useEffect(() => {
    setCount(prev => prev + 1);
    setCount(prev => prev + 1);
  }, []);
  console.log('render:', count);
options:
  - "render: 0"
  - "render: 1"
  - "render: 2"
  - "render: 0 (초기 렌더) → render: 2 (재렌더)"
answer: "render: 0 (초기 렌더) → render: 2 (재렌더)"
explanation: "useState의 setter에 함수를 전달하면 batching 후 한 번에 반영됩니다. 초기 렌더 시 count는 0이고, useEffect 후 2로 업데이트되어 재렌더됩니다."
xp: 6
```
````

#### 유형 C: 버그 찾기

#### 유형 D: 빈칸 선택형 (Word Bank 방식)

**개요**: 코드의 빈칸에 들어갈 토큰을 보기 풀(Word Bank)에서 선택한다. 기존 빈칸 채우기(직접 타이핑)의 쉬운 버전.

**동작**:

### 2-6. 스페이스드 리피티션 (복습) 시스템

**개요**: 말해보카의 망각곡선 기반 복습 시스템을 적용하여 완료한 Phase의 핵심 개념을 최적 타이밍에 복습한다.

**망각곡선 복습 주기**:

| 학습 후 경과 | 복습 제안 |
|-------------|----------|
| 1일 | 첫 번째 복습 |
| 3일 | 두 번째 복습 |
| 7일 | 세 번째 복습 |
| 14일 | 네 번째 복습 |
| 30일 | 다섯 번째 복습 |

**동작**:
1. Phase 완료 시 해당 Phase의 핵심 문제들이 복습 큐에 등록됨
2. 사이트 접속 시 복습 예정 문제가 있으면 팝업 알림:
   "📚 Phase 1: React 기초 복습이 준비되었습니다! (5문제)"
3. 복습은 해당 Phase의 연습 문제를 랜덤 셔플하여 5~10문제 출제
4. 정답률에 따라 다음 복습 간격 조정:
   - 90%+ 정답: 다음 간격으로 이동 (예: 3일 → 7일)
   - 70~89% 정답: 같은 간격 유지
   - 70% 미만: 이전 간격으로 되돌림 (예: 7일 → 3일)

**localStorage 데이터 구조 확장**:
```json

      "completedAt": "2026-03-09",

      "interval": 1,
      "reviewCount": 0
    }
  }
}
```

**복습 문제 수집 메커니즘**:
- `PHASE_MAP`을 확장하여 각 Phase별 소속 문서 경로 목록을 포함:
  ```javascript
  'phase-01-react-basics': {

      '/phase-01-react-basics/03-props-and-state',
      '/phase-01-react-basics/04-hooks-deep-dive',
    ]
  }
  ```
- 복습 시작 시 해당 Phase의 pages를 순회하며 각 문서 내 `exercise`/`quiz` 블록을 DOM에서 파싱
- 수집된 문제를 랜덤 셔플하여 5~10문제 출제

**UI**: 사이드바 상단에 복습 알림 뱃지

```
┌─────────────────────┐
│  🔥 12일 연속 학습!   │
```

### 2-10. 즉각 피드백 강화

## 5. 마크다운 콘텐츠 변경 전략

### 5-1. 기존 콘텐츠와의 공존

- 기존 `quiz` 코드 블록은 그대로 유지 (하위 호환)
- 신규 `exercise` 코드 블록을 각 H2 섹션 사이에 추가
- 레슨 모드 OFF 시 기존처럼 전체 스크롤 + 퀴즈 방식으로 동작
- 레슨 모드 ON 시 H2 단위 카드 + exercise 인터리빙 방식으로 동작

### 5-2. 연습 문제 추가 기준

| Phase | 추가할 exercise 수 (문서당) | 주요 연습 유형 |
|-------|--------------------------|---------------|
| Phase 0 (JS/TS) | 3~4개 | 코드 조립, 출력 예측, word-bank |
| Phase 1 (React) | 3~4개 | 코드 조립, 버그 찾기, 출력 예측 |
| Phase 2 (환경) | 1~2개 | MCQ, 개념 분류 |
| Phase 3 (컴포넌트) | 4~5개 | 코드 조립, word-bank, 개념 분류 |
| Phase 4 (네비게이션) | 3~4개 | 코드 조립, 매칭, word-bank |
| Phase 5 (상태관리) | 3~4개 | 출력 예측, 코드 조립, 버그 찾기 |
| Phase 6 (New Arch) | 2~3개 | MCQ, 개념 분류, 매칭 |
| Phase 7 (네이티브) | 2~3개 | 코드 조립, 버그 찾기, word-bank |

| Phase 9 (배포) | 1~2개 | MCQ, 개념 분류 |

## 7. 리스크 및 대응

| 리스크 | 영향 | 대응 |
|--------|------|------|
| 과도한 게이미피케이션으로 학습 집중도 저하 | 게임에만 몰두하고 실제 학습 소홀 | 레슨 모드 OFF 옵션 제공, XP는 실제 학습 활동에만 부여 |
| 하트 시스템에 대한 사용자 피로감 | 학습 중단 유발 | 하트 시스템 OFF 옵션, 복습으로 하트 회복 경로 제공 |
| Phase 순차 해금으로 자유도 제한 | 특정 Phase만 학습하고 싶은 사용자 불만 | "자유 모드" 토글로 전체 해금 가능하게 설정 |
| localStorage 데이터 용량 증가 | 5~10MB 한계 접근 | 일별 XP history 30일 초과분 정리, 압축 |
| 드래그 앤 드롭 모바일 호환성 | 터치 환경에서 드래그 어려움 | 터치 환경 감지 시 "탭하여 선택 → 탭하여 배치" 방식으로 전환 |
| 연습 문제 100개+ 콘텐츠 작성 부담 | 구현은 되었으나 콘텐츠 부족 | Phase별 우선순위 두고 점진적 추가, 핵심 Phase (0, 1, 3, 5) 우선 |
| 사운드로 인한 사용자 불편 | 공공장소 사용 시 민망 | 사운드 기본 OFF, 설정에서 ON 가능 |
